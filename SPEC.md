# Note a Note

## 概要

作曲アイデアをすばやく記録・再生できる音楽メモアプリ。
テキスト・ピアノロール譜面・コードブロックを自由に組み合わせたNotionライクなブロック構成で、思いついた歌詞やメロディーをその場でメモし、すぐに再生して確認できる。耳コピのメモや作曲のスケッチに特化したシンプルな設計。

## ターゲットユーザー

- 作曲・DTMに興味があり、思いついたメロディーや歌詞をすぐ記録したい人
- 楽器演奏者で耳コピのメモを手軽に取りたい人
- 本格的なDAWを使うほどではないが、音楽的なアイデアを残しておきたい人
- 主に個人利用（当面は自分自身がメインユーザー）

## コスト設計

### 使用API・サービス一覧

| サービス | 用途 | 備考 |
|----------|------|------|
| Expo（EAS Build） | ビルド・配布 | 個人利用は無料枠あり |
| expo-av / expo-audio | 音声再生 | ローカルライブラリ、API不要。mixWithOthersモード使用 |
| AsyncStorage または expo-sqlite | ローカルデータ保存 | ローカルのみ、外部サービス不要 |

### 想定月額コスト

**ほぼ無料（$0〜数ドル）**

- 外部APIを一切使用しないため、ランニングコストなし
- EAS Buildは個人プラン（無料）で月30ビルドまで対応

### 無料枠・代替手段

- 音源はexpo-avでローカル再生（Web Audio API相当）、外部音源サービス不要
- ストレージはデバイスローカルのみのため、サーバー費用なし

## アーキテクチャ概要

### 構成図

```
[UI Layer]
  ├── メモ一覧画面
  └── メモ編集画面
        ├── TextBlock（テキスト入力）
        ├── PianoRollBlock（ピアノロール譜面）
        │     ├── 鍵盤UI（タップで音配置）
        │     └── ノートバー（ドラッグで音長調整）
        └── ChordBlock（コード選択・再生）
              └── 2段階ピッカー（ルート音 × コードタイプ）

[State Layer]
  └── Zustand Store
        ├── メモ一覧・編集状態
        ├── 再生状態（BPM、再生中フラグ）
        └── ブロックデータ

[Audio Layer]
  └── AudioEngine（expo-av wrapper）
        ├── シーケンサー（BPMに従いノートを順次再生）
        └── シンセ音源エンジン（固定音色・単音／和音再生）

[Data Layer]
  └── AsyncStorage / SQLite
        └── メモJSON（ブロック配列を直列化して保存）
```

### データの流れ

1. ユーザーが鍵盤をタップ → ノートデータ（音高・位置）をPianoRollBlockのstateに追加
2. ドラッグ操作 → ノートの長さ（duration）を更新
3. 再生ボタン → AudioEngineがBPMを元にノートを順次スケジューリングして再生
4. コードブロックをタップ → コードに対応する和音ノートを即時再生
5. 保存 → メモ全体（ブロック配列）をJSONシリアライズしてローカルに書き込み

## 非機能要件

- **パフォーマンス**：メモ一覧は100件程度まで遅延なく表示できること
- **オフライン**：全機能がオフラインで動作すること（外部通信なし）
- **データ永続性**：アプリ再起動後もメモが保持されること
- **対応OS**：iOS 16以上 / Android 13以上
- **UX**：メモ新規作成〜ノート入力〜再生までが3タップ以内で完結すること
- **オーディオ共存**：SpotifyなどBGM再生中でもアプリの音を重ねて再生できること（expo-audio の `mixWithOthers` モード使用）
